**Commit Rule**
-----------------------------------------------------

> feat: 기능 추가

> fix: 버그 수정

    - success
    - fail

> docs: 문서 수정

> style: 코드 말고 들여쓰기나 세미콜론 같은 경우

> refactor: 코드 리펙토링

> test: 테스트 코드 수정

> chore: 코드 말고 설정 변경
